# 実装機能一覧 (Features)

ポモドーロタイマーアプリケーションで実装する機能のリストです。

## 1. プロジェクト基盤
- **Flask アプリケーション構造**: `create_app()` ファクトリパターンを採用したバックエンド構成。
- **依存管理**: `requirements.txt` によるパッケージ管理。
- **環境設定**: 開発/テスト/本番環境の切り替え対応。

## 2. ユーザーインターフェース (UI)
- **メイン画面**: 添付モックに基づいたデザイン。
- **円形プログレスバー**: 残り時間を視覚的に示す SVG/Canvas リング。
- **タイマー表示**: 分:秒 形式の大きなデジタル時計。
- **操作ボタン**: 開始(Start)、一時停止(Pause)、リセット(Reset) ボタン。
- **進捗パネル**: 今日の完了セッション数と合計集中時間の表示。

## 3. タイマーロジック (TimerEngine)
- **純粋ロジック**: DOM に依存しない独立した JS モジュール。
- **正確な計時**: `Date.now()` (または `performance.now()`) の差分計算によるドリフト（時間のズレ）防止。
- **状態管理**: 作業中、休憩中、停止中などの状態遷移。
- **テスト容易性**: 時間依存（`clock`）を注入可能な設計。

## 4. クライアント機能 (UIController & Storage)
- **DOM 操作**: タイマーイベントに応じた画面更新。
- **ローカル永続化**: `localStorage` を使用し、リロードやブラウザ再起動後もタイマー状態を復元。
- **オフライン対応**: ネットワーク切断時に完了したセッションを保持し、再接続時に送信するキュー機能。

## 5. サーバー API (Backend)
- **セッション管理 API**:
    - `POST /api/session/start`: 作業開始の記録。
    - `POST /api/session/complete`: 作業完了の記録（実績時間）。
    - `POST /api/session/cancel`: 作業中断の記録。
- **統計 API**:
    - `GET /api/stats/today`: 当日の完了回数と合計時間の取得。

## 6. データ永続化 (Database)
- **データベース**: SQLite (開発用) / 将来的な PostgreSQL 対応。
- **ORM**: SQLAlchemy を使用したモデル定義 (`Session` モデル)。
- **リポジトリパターン**: DB アクセスを抽象化し、テスト時のモック差し替えを容易にする。

## 7. 通知・サウンド
- **ブラウザ通知**: タイマー終了時にデスクトップ通知を表示。
- **サウンド再生**: 終了時のアラート音。
- **設定**: 通知・サウンドの ON/OFF 切り替え（Notifier 抽象化）。

## 8. 設定機能
- **時間設定**: 作業時間（デフォルト25分）、短休憩、長休憩の時間をカスタマイズ可能。
- **設定保存**: 設定内容のローカル保存（またはサーバー保存）。

## 9. テスト & 品質保証
- **ユニットテスト (Frontend)**: `vitest` + `jsdom` による `TimerEngine` のロジックテスト。
- **ユニットテスト (Backend)**: `pytest` + `in-memory SQLite` による API/モデルのテスト。
- **CI (継続的インテグレーション)**: GitHub Actions での自動テスト実行。

## 10. デプロイ & 運用
- **コンテナ化**: Dockerfile の作成。
- **Web サーバー**: Gunicorn + Nginx 構成（本番想定）。
- **ロギング**: アプリケーションログの出力設定。
