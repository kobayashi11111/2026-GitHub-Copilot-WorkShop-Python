# データモデル仕様

## 概要

ポモドーロタイマーアプリケーションのデータモデルは、JSON形式でファイルに永続化されます。すべてのゲーミフィケーション要素（XP、レベル、バッジ、ストリーク、セッション履歴）がこのデータモデルで管理されます。

## データファイル

### ファイル情報

- **ファイル名**: `pomodoro_data.json`
- **場所**: アプリケーション実行ディレクトリ
- **エンコーディング**: UTF-8
- **フォーマット**: JSON（整形あり、インデント2スペース）

## データ構造

### ルートオブジェクト

```json
{
  "xp": 0,
  "level": 1,
  "total_pomodoros": 0,
  "completed_dates": [],
  "badges": [],
  "sessions": [],
  "current_streak": 0,
  "best_streak": 0,
  "last_completion_date": null
}
```

## フィールド定義

### xp (number)

**説明**: ユーザーが獲得した累計経験値

**型**: `number` (float)

**初期値**: `0`

**更新タイミング**: ポモドーロ完了時

**計算式**: `xp += (作業時間分 / 25) * 100`

**例**:
```json
"xp": 1550.0
```

---

### level (number)

**説明**: 現在のユーザーレベル

**型**: `number` (integer)

**初期値**: `1`

**更新タイミング**: ポモドーロ完了時（XP増加後に自動計算）

**計算式**: `level = 1 + floor(xp / 500)`

**例**:
```json
"level": 4
```

**レベルとXPの対応表**:

| レベル | 必要XP（下限） | 必要XP（上限） |
|--------|---------------|---------------|
| 1      | 0             | 499           |
| 2      | 500           | 999           |
| 3      | 1000          | 1499          |
| 4      | 1500          | 1999          |
| 5      | 2000          | 2499          |
| ...    | ...           | ...           |

---

### total_pomodoros (number)

**説明**: 完了したポモドーロセッションの累計回数

**型**: `number` (integer)

**初期値**: `0`

**更新タイミング**: ポモドーロ完了時（+1）

**用途**: 
- ステータス表示
- 回数達成バッジの判定基準

**例**:
```json
"total_pomodoros": 127
```

---

### completed_dates (array)

**説明**: ポモドーロを完了した日付のリスト（重複なし）

**型**: `array` of `string` (YYYY-MM-DD形式)

**初期値**: `[]`

**更新タイミング**: ポモドーロ完了時（その日初回のみ追加）

**用途**: ストリーク計算、統計情報

**例**:
```json
"completed_dates": [
  "2026-02-18",
  "2026-02-19",
  "2026-02-20",
  "2026-02-24"
]
```

---

### badges (array)

**説明**: 獲得したバッジのIDリスト

**型**: `array` of `string`

**初期値**: `[]`

**更新タイミング**: ポモドーロ完了時（条件を満たした場合）

**バッジIDの形式**:
- ストリークバッジ: `streak_{日数}`
- 回数達成バッジ: `count_{回数}`

**例**:
```json
"badges": [
  "streak_3",
  "streak_7",
  "count_10",
  "count_25",
  "count_50"
]
```

**バッジID一覧**:

| バッジID | 条件 | 表示名 |
|----------|------|--------|
| `streak_3` | 3日連続 | 🔥 3日連続 |
| `streak_7` | 7日連続 | 🔥🔥 7日連続 |
| `streak_30` | 30日連続 | 🔥🔥🔥 30日連続 |
| `streak_100` | 100日連続 | 🔥🔥🔥🔥 100日連続 |
| `count_10` | 10回達成 | ⭐ 10回達成 |
| `count_25` | 25回達成 | ⭐⭐ 25回達成 |
| `count_50` | 50回達成 | ⭐⭐⭐ 50回達成 |
| `count_100` | 100回達成 | ⭐⭐⭐⭐ 100回達成 |
| `count_250` | 250回達成 | ⭐⭐⭐⭐⭐ 250回達成 |

---

### sessions (array)

**説明**: 完了したポモドーロセッションの詳細履歴

**型**: `array` of `object`

**初期値**: `[]`

**更新タイミング**: ポモドーロ完了時（新しいセッションを追加）

**用途**: 統計情報の集計、日別グラフの生成

**セッションオブジェクトの構造**:

```json
{
  "date": "2026-02-24",
  "datetime": "2026-02-24T10:30:00.123456",
  "duration": 25,
  "xp_earned": 100
}
```

**セッションフィールド**:

| フィールド | 型 | 説明 |
|-----------|---|------|
| `date` | string | 日付（YYYY-MM-DD形式） |
| `datetime` | string | 完了日時（ISO8601形式） |
| `duration` | number | 作業時間（分） |
| `xp_earned` | number | 獲得したXP |

**例**:
```json
"sessions": [
  {
    "date": "2026-02-23",
    "datetime": "2026-02-23T09:15:30.456789",
    "duration": 25,
    "xp_earned": 100
  },
  {
    "date": "2026-02-23",
    "datetime": "2026-02-23T10:45:12.234567",
    "duration": 25,
    "xp_earned": 100
  },
  {
    "date": "2026-02-24",
    "datetime": "2026-02-24T14:20:05.678901",
    "duration": 25,
    "xp_earned": 100
  }
]
```

---

### current_streak (number)

**説明**: 現在の連続実行日数

**型**: `number` (integer)

**初期値**: `0`

**更新タイミング**: ポモドーロ完了時

**更新ロジック**:
1. 初回: `1` に設定
2. 前回と同日: 変更なし
3. 前回から1日後: `+1`
4. 前回から2日以上後: `1` にリセット

**例**:
```json
"current_streak": 7
```

---

### best_streak (number)

**説明**: 過去最長の連続実行日数記録

**型**: `number` (integer)

**初期値**: `0`

**更新タイミング**: ストリーク更新時（current_streakがbest_streakを超えた場合）

**例**:
```json
"best_streak": 15
```

---

### last_completion_date (string | null)

**説明**: 最後にポモドーロを完了した日付

**型**: `string` (YYYY-MM-DD形式) または `null`

**初期値**: `null`

**更新タイミング**: ポモドーロ完了時

**用途**: ストリーク判定の基準日

**例**:
```json
"last_completion_date": "2026-02-24"
```

## データ操作

### 初期化

新規ユーザーまたはファイルが存在しない場合、以下のデフォルトデータで初期化されます：

```json
{
  "xp": 0,
  "level": 1,
  "total_pomodoros": 0,
  "completed_dates": [],
  "badges": [],
  "sessions": [],
  "current_streak": 0,
  "best_streak": 0,
  "last_completion_date": null
}
```

### 保存

**トリガー**: `GamificationManager.add_pomodoro()` メソッド内で自動保存

**処理フロー**:
1. データの更新（XP、レベル、セッション追加等）
2. `save_data()` メソッド呼び出し
3. JSON形式でファイルに書き込み

**保存設定**:
- `ensure_ascii=False`: 日本語などの非ASCII文字をそのまま保存
- `indent=2`: 可読性のため2スペースインデント

### 読み込み

**タイミング**: `GamificationManager` インスタンス化時

**エラーハンドリング**:
- ファイルが存在しない → デフォルトデータで初期化
- JSON解析エラー → デフォルトデータで初期化
- IOエラー → デフォルトデータで初期化

## データ整合性

### 保証される整合性

1. **XPとレベルの整合性**: レベルアップチェックは常にXP増加後に実行
2. **ストリークの一貫性**: 同一セッション内では一度だけ更新
3. **日付の一意性**: `completed_dates` は重複を含まない
4. **バッジの一意性**: 同じバッジIDは一度だけ追加

### 潜在的な問題と対策

**問題**: 複数プロセスでの同時アクセス
**対策**: 現在は単一プロセスでの使用を想定（将来的にはファイルロックを検討）

**問題**: 日付/時刻の変更
**対策**: システム日時を信頼（NTPによる時刻同期を推奨）

**問題**: 手動編集によるデータ破損
**対策**: JSONスキーマバリデーション（将来的な拡張）

## データマイグレーション

現在のバージョンではデータマイグレーション機能はありませんが、将来的な拡張として以下が考えられます：

### 想定される拡張

1. **バージョン番号の追加**: `"schema_version": 1`
2. **マイグレーションスクリプト**: バージョン間の変換処理
3. **バックアップ機能**: データ更新前の自動バックアップ

## セキュリティとプライバシー

### 保存データの性質

- **個人情報**: 含まれない（名前、メールアドレス等なし）
- **機密情報**: 含まれない
- **使用統計**: タイムスタンプと作業時間のみ

### 推奨事項

- ファイルパーミッション: ユーザーのみ読み書き可能（600）
- バックアップ: 定期的なバックアップを推奨
- バージョン管理: データファイルをgitで管理しないこと
