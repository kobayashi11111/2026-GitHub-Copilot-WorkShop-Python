## はじめに

### ポモドーロタイマーとは

ポモドーロ・テクニックは、時間管理手法の一つです：

1. **25分間集中して作業**（1ポモドーロ）
2. **5分間休憩**
3. これを繰り返す

このアプリは、ポモドーロ・テクニックにゲーミフィケーション要素を加えることで、継続的な利用とモチベーション維持をサポートします。

### このアプリの特徴

- ⏰ **シンプルなタイマー機能**
- 📊 **経験値(XP)とレベルシステム**
- 🏆 **達成バッジコレクション**
- 🔥 **連続日数(ストリーク)記録**
- 📈 **詳細な統計情報**
- 💾 **自動データ保存**

## インストールと起動

### 必要な環境

- Python 3.x
- 標準ライブラリのみ（追加インストール不要）

### 起動方法

```bash
# ディレクトリに移動
cd 1.pomodoro

# アプリを起動
python app.py
```

初回起動時、`pomodoro_data.json` ファイルが自動的に作成されます。

### 1. ポモドーロを開始する

**操作**: メニューで `1` を選択

   ==================================================

3. 完了すると結果が表示される

- **合計ポモドーロ**: これまでに完了した回数

- **最長ストリーク**: 過去最高の連続記録

## ゲーミフィケーション要素

### 経験値(XP)システム

#### XPの獲得

- **25分のポモドーロ完了**: +100 XP
- **カスタム時間**: `(時間（分） / 25) × 100` XP
  - 例: 50分完了 → +200 XP

#### レベルアップ

- **レベル1 → レベル2**: 500 XP（5回のポモドーロ）
- **レベル2 → レベル3**: 1000 XP（累計10回のポモドーロ）
- **レベル3 → レベル4**: 1500 XP（累計15回のポモドーロ）
- ...

**計算式**: レベル = 1 + (XP ÷ 500)

**レベルアップ時の特典**:
- 「レベルN到達！」バッジが表示される
- 達成感を得られる

### ストリーク（連続日数）

#### ストリークとは

毎日ポモドーロを実行することで、連続日数がカウントされます。

#### ストリークの更新ルール

- **初日**: ストリーク = 1日
- **翌日に実行**: ストリーク +1
- **2日以上空く**: ストリーク = 1 にリセット
- **同じ日に複数回**: ストリークは変わらない

#### ベストストリーク

これまでの最長連続日数が記録されます。ストリークがリセットされても、ベストストリークは保持されます。

#### ストリークのコツ

- 毎日最低1回はポモドーロを実行する
- 1日の最初に実行すると忘れない
- 週末も継続することが重要

## 統計とバッジ

### バッジシステム

#### ストリークバッジ（連続日数）

| バッジ | 条件 | 説明 |
|--------|------|------|
| 🔥 3日連続 | 3日連続でポモドーロ実行 | 継続の第一歩 |
| 🔥🔥 7日連続 | 7日連続（1週間） | 習慣化の証 |
| 🔥🔥🔥 30日連続 | 30日連続（1ヶ月） | 真の継続力 |
| 🔥🔥🔥🔥 100日連続 | 100日連続 | 圧倒的な継続力 |

#### 回数達成バッジ

| バッジ | 条件 | 説明 |
|--------|------|------|
| ⭐ 10回達成 | 累計10回 | 最初のマイルストーン |
| ⭐⭐ 25回達成 | 累計25回 | 本格的な利用者 |
| ⭐⭐⭐ 50回達成 | 累計50回 | ベテラン利用者 |
| ⭐⭐⭐⭐ 100回達成 | 累計100回 | マスターレベル |
| ⭐⭐⭐⭐⭐ 250回達成 | 累計250回 | レジェンド |

#### バッジ獲得のコツ

1. **毎日継続する**: ストリークバッジを狙う
2. **コツコツ積み重ねる**: 回数バッジは自然に獲得できる
3. **複数のバッジを同時獲得**: 条件を満たすと同時に複数獲得できる

### 統計の活用

#### 週間統計の見方

週間統計を見ることで：
- 今週の生産性を把握
- 曜日ごとの傾向を確認
- 1日の目標回数を設定

**おすすめの使い方**:
1. 毎週月曜日に前週の統計を確認
2. 今週の目標を設定（例: 1日平均3回）
3. 週末に達成状況を確認

#### 月間統計の見方

月間統計を見ることで：
- 今月の全体的な傾向を把握
- 長期的な成長を確認
- モチベーションの維持

**おすすめの使い方**:
1. 月初に前月の統計を確認
2. 今月の目標を設定（例: 月間60回）
3. 月末に振り返り

## よくある質問

### Q1: ポモドーロの時間を変更できますか？

現在のバージョンでは、作業時間は25分、休憩時間は5分に固定されています。

将来的には設定機能を追加予定です。

### Q2: データはどこに保存されますか？

`pomodoro_data.json` というファイルに、アプリを実行したディレクトリに保存されます。

### Q3: データをバックアップするには？

`pomodoro_data.json` ファイルをコピーするだけでバックアップできます。

```bash
# バックアップを作成
cp pomodoro_data.json pomodoro_data_backup.json

# リストアする
cp pomodoro_data_backup.json pomodoro_data.json
```

### Q4: データをリセットするには？

`pomodoro_data.json` ファイルを削除して、アプリを再起動してください。

```bash
# データを削除
rm pomodoro_data.json

# アプリを起動すると新しいデータファイルが作成される
python app.py
```

### Q5: ストリークがリセットされた！

ストリークは、前日にポモドーロを実行していない場合にリセットされます。

ベストストリーク（最長記録）は保持されるので、また挑戦しましょう！

### Q6: 休憩時間をスキップできますか？

はい。休憩の確認画面で `n` を入力すると、休憩をスキップしてメニューに戻ります。

### Q7: 複数のデバイスでデータを共有できますか？

現在のバージョンでは、デバイス間の同期機能はありません。

`pomodoro_data.json` ファイルを手動でコピーすることで、データを移行できます。

### Q8: 何回のポモドーロを目指すべきですか？

個人の作業スタイルによりますが、一般的な目安：

- **初心者**: 1日2-3回
- **中級者**: 1日5-7回
- **上級者**: 1日10回以上

無理せず、自分のペースで継続することが重要です。

## トラブルシューティング

### 問題: アプリが起動しない

**原因**: Pythonがインストールされていない

**解決方法**:
```bash
# Pythonのバージョンを確認
python --version

# または
python3 --version
```

Python 3.x がインストールされていることを確認してください。

---

### 問題: データが保存されない

**原因1**: ファイルの書き込み権限がない

**解決方法**:
```bash
# ディレクトリの権限を確認
ls -la

# 権限を変更
chmod u+w .
```

**原因2**: ディスクの空き容量がない

**解決方法**: 不要なファイルを削除して、空き容量を確保してください。

---

### 問題: タイマーが正確に動かない

**原因**: システムの負荷が高い

**解決方法**:
- 他のアプリを閉じる
- バックグラウンドプロセスを減らす

**注意**: 
- このアプリは `time.sleep()` を使用しているため、システムの状態により若干の誤差が生じる可能性があります
- 誤差は通常数秒以内です

---

### 問題: JSONDecodeError が表示される

**原因**: データファイルが破損している

**解決方法**:
```bash
# バックアップがある場合
cp pomodoro_data_backup.json pomodoro_data.json

# バックアップがない場合（データをリセット）
rm pomodoro_data.json
python app.py
```

---

### 問題: バッジが獲得できない

**確認事項**:
1. **ストリークバッジ**: 連続日数を確認（ステータス画面で確認）
2. **回数バッジ**: 累計回数を確認（ステータス画面で確認）
3. **重複獲得**: 同じバッジは2回獲得できません

**デバッグ方法**:
```bash
# データファイルの内容を確認
cat pomodoro_data.json

# 整形して見やすく
python -m json.tool pomodoro_data.json
```
